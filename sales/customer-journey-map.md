# ODE Customer Journey Map

## Journey Overview

Two parallel journeys feeding a shared platform:

```
PLG:  Discover → Sign Up → Activate → Convert → Expand → Advocate
SLG:  Identify → Engage → Pilot → Contract → Expand → Advocate
```

---

## Stage 1: Awareness

### PLG Path
| Element | Detail |
|---------|--------|
| **Touchpoints** | Google search, LinkedIn post, blog article, GitHub, conference talk |
| **Content** | "AI-native engineering platform" positioning, technical blog posts, demo videos |
| **User Mindset** | "There must be a better way to do simulation" |
| **Actions** | Visit website, watch demo video, read blog post |
| **Key Metric** | Website unique visitors (target: 5,000/mo by Q4) |

### SLG Path
| Element | Detail |
|---------|--------|
| **Touchpoints** | ABM outreach, LinkedIn InMail, conference meeting, referral |
| **Content** | Personalized outreach, industry-specific one-pagers |
| **User Mindset** | "Our tools are slowing us down" |
| **Actions** | Open email, visit website, accept meeting |
| **Key Metric** | Response rate (target: >15% for Tier 1 accounts) |

## Stage 2: Consideration

### PLG Path

| **User Mindset** | "Can this work for our specific use case?" |

| **Key Metric** | Demo → pilot conversion (target: 30%) |

## Stage 4: Onboarding

| Day | Milestone | Content/Action |
|----:|----------|---------------|
| 0 | Sign up | Welcome email, guided tour |

| 3 | Explore tools | "6 tools" feature email |
| 7 | Power user tips | Advanced features email |

### SLG Path (30-Day Pilot)
| Week | Milestone | Content/Action |
|-----:|----------|---------------|
| 1 | Setup | Workspace configured, team access, success criteria confirmed |
| 1-2 | First project | Champion builds real project with CSM support |
| 2-3 | Team adoption | Team members onboarded, 3+ users active |
| 3-4 | Results review | Usage data, success criteria evaluation, ROI documentation |

## Stage 5: Adoption

**Key Activation Moments** (signals of healthy adoption):

| Moment | Significance | Metric |
|--------|-------------|--------|
| First AI model generated | User sees core value | Time to first model <10 min |
| First simulation run | User validates AI output | Day 1-3 |
| Third project created | Regular usage established | Week 2-4 |
| First team invite | Viral/expansion signal | Any time |
| First AI token purchase | Willing to pay for AI value | Week 2-4 |
| First compute-heavy sim | Power user behavior | Month 1-2 |

---

## Stage 6: Expansion

**Expansion Triggers:**

| Trigger | Signal | Action |
|---------|--------|--------|
| Hit 80% AI token limit | Usage growing | In-app upgrade prompt + email |
| Invite 3+ team members | Team adoption | "Upgrade to Team" prompt |
| 5+ active projects | Power user | Account review, suggest Pro/Team |
| Request advanced features (SSO, API) | Enterprise need | Route to sales |
| Cross-department interest | Org-wide potential | Enterprise expansion discussion |

**Expansion Playbook:**
1. Monitor usage dashboards weekly for expansion signals
2. Proactive outreach when usage hits 70% of plan limits
3. Quarterly business review for Team/Enterprise accounts
4. Success story collection for internal champions to share

## Churn Risk Signals & Intervention

| Risk Signal | Threshold | Intervention |
|-------------|-----------|-------------|
| Login drop-off | No login in 7 days (paid) | Automated re-engagement email |
| Usage decline | <50% of prior month usage | Account owner check-in |
| Support tickets spike | 3+ tickets in a week | Priority support + PM review |
| Failed simulations | >50% failure rate | Technical support outreach |
| Competitor evaluation | Visits competitor pages (if tracked) | Value reinforcement call |
| Champion leaves | Key user account deactivated | Identify new champion, executive outreach |
| Contract renewal approaching | 60 days before renewal | QBR with usage data + ROI summary |

### Churn Intervention Sequence
1. **Day 7 inactive**: Automated "We miss you" email with new feature highlights
2. **Day 14 inactive**: Personal email from account owner with specific value reminder
3. **Day 21 inactive**: Phone/video call attempt with offer to help with specific project
4. **Day 30 inactive**: Executive outreach with retention offer (if high-value account)
