# PRD: ODE G2M Strategy v2 — "Dassault of the AI Era"

**Status:** Draft  
**Owner:** CEO  
**Date:** 2026-04-12  
**Team:** 6 core + 2 GTM (AI Marketing × 0.5 + Sales × 1)

---

## Problem Statement

ODE 目前的 G2M 定位聚焦于"AI-native engineering platform"，过于通用，缺乏历史参照锚点，难以在工程师和企业决策者中形成即时的认知共鸣。

竞争对手达索系统（Dassault Systèmes）用 30 年构建了工业界的"全栈工程平台"标准——CAD（CATIA）、仿真（SIMULIA）、PLM（ENOVIA）、MBSE（GENESYS）覆盖全部工程学科。但它的核心弱点是：架构 30 年未重构，AI 为后期缝合，许可证价格让 99% 的工程师和中小企业望而却步。

ODE 拥有达索无法快速复制的两项资产：**AI-native 架构** 和 **全学科工具覆盖**（CAD、MBSE、System Simulation、FEM、CFD、Scientific Computing、Reinforcement Learning、KiCad 电路设计、Multi-objective Optimisation）。

第二个核心问题是销售执行力不足：当前 6 人团队无专职市场和销售，CEO 兼任销售，是增长的最大瓶颈。

---

## Solution

### 定位重构：Orthogonal = AI 时代的达索

**从：** "AI-native engineering platform"（功能描述）  
**到：** "Orthogonal — the Dassault of the AI era"（历史坐标 + 竞争宣言）

这一定位传递三个信息：
1. 达索的参照让每个工业工程师立刻理解 ODE 的野心与全栈性
2. "AI era"明确代际断层——不是改良，是替代
3. Orthogonal（母品牌）承载平台级定位，ODE 成为旗舰产品线

ODE 平台工具全覆盖（对标达索各模块）：

| ODE Component | CATIA Systems | AI-native Modelica 多域物理建模 |
| ODE Geometry | CATIA | 参数化 CAD，浏览器端 |
| ODE System | GENESYS | MBSE + SysML 系统架构 |
| ODE Vision | SIMULIA 3DX | 3D 实时仿真可视化 |
| ODE Paper | ENOVIA | 协作文档 + 需求追溯 |
| ODE Interaction | 3DEXPERIENCE | 实时参数探索 |
| **ODE FEM** | SIMULIA Abaqus | 有限元分析，AI 辅助网格与求解 |
| **ODE CFD** | SIMULIA XFlow | 计算流体力学 |

### GTM 团队补强

**新增 1.5 个 GTM 角色：**

## User Stories

### 品牌与定位

1. 作为工程决策者，我想一眼看懂 ODE 的野心，以便快速判断是否值得深入了解。
2. 作为工程师，我想用一句话向同事解释 ODE，以便引入团队评估。
3. 作为投资人，我想看到清晰的"类达索"叙事，以便理解 $1B+ 的潜在天花板。
4. 作为 CAD 经销商，我想了解 ODE 如何补充而非替代我的产品线，以便评估合作机会。

### 工具覆盖

5. 作为 FEM 工程师，我想在 ODE 内完成有限元建模与分析，以便不切换到 ANSYS 或 Abaqus。
6. 作为 CFD 工程师，我想在同一平台完成 CAD + CFD 仿真，以便消除文件导入导出的麻烦。
7. 作为电子工程师，我想用 ODE Circuit（KiCad）设计 PCB 并与机械系统联仿，以便实现真正的机电一体化设计。
8. 作为控制工程师，我想用 ODE RL 训练控制策略，以便替代 Simulink + Python 的分裂工作流。
9. 作为系统工程师，我想运行多目标优化，以便在重量、成本、性能之间找到 Pareto 最优解。
10. 作为科研人员，我想在 ODE Compute 中运行数值计算并可视化，以便替代 MATLAB/Python + 额外可视化工具。

### GTM 执行

11. 作为 AI Marketing Specialist，我想使用 AI 工具每周产出 5+ 件内容，以便以 0.5 人力建立完整内容矩阵。
12. 作为 Sales Manager，我想建立 3-5 个渠道合作伙伴关系，以便在不增加直销人员的情况下扩大覆盖范围。
13. 作为 CEO，我想把大客户销售时间从 60% 降至 30%，以便专注产品与融资。
14. 作为渠道合作伙伴（工程咨询公司），我想获得 ODE 的产品培训和销售支持，以便向客户推荐。
15. 作为潜在企业客户，我想看到清晰的 ROI 案例（ODE vs. 达索/ANSYS），以便向采购委员会汇报。

### 设计系统（Apple 风格）

16. 作为设计师，我想使用统一的 Apple 风格设计规范，以便所有营销材料和产品 UI 保持一致的高级感。
17. 作为用户，我想看到极简、克制的界面设计，以便专注于工程内容而不被 UI 干扰。

## Implementation Decisions

### 定位执行

- **主品牌**：Orthogonal（orthogonal.dev）承载"AI 时代的达索"叙事
- **产品品牌**：ODE 作为 Orthogonal 旗舰工程平台
- **核心口号**：
  - 主：*"The Dassault of the AI era."*（战略层）
  - 副：*"Engineering reimagined with AI."*（产品层）
  - 技术层：*"One platform. Every discipline. AI-native."*
- **竞争叙事更新**：所有竞品对比从达索切入，建立参照系

### 工具发布节奏

**Phase 1（现有）**：Component, Paper, System, Vision, Geometry, Interaction  
**Phase 2（Q2-Q3 2026）**：ODE FEM, ODE CFD  
**Phase 3（Q4 2026）**：ODE Compute, ODE Optim  
**Phase 4（2027）**：ODE RL, ODE Circuit（KiCad）

每个新工具发布都是一次独立的内容营销事件，配合"vs. 达索对应模块"的比较内容。

### 团队结构（更新后）

```
CEO                    — 战略、融资、重点大客户（20-30% 时间）
CTO                    — 技术、产品、AI 架构
Designer               — 产品 UI + 品牌视觉（Apple 设计系统）

Backend Engineer       — 产品开发、API

```

### 渠道策略（更新）

**直销**（Sales Manager 负责）：
- ICP：欧洲 + 北美工程 OEM，50-500 人工程团队
- 方法：LinkedIn ABM + 技术 Demo + 30 天 Pilot
- 目标：Year 1 签约 8-12 家企业客户，$400K ARR

**分销渠道**（Sales Manager 负责）：
- 渠道类型：工程咨询公司（Tier 1）、CAD 经销商（Tier 2）、系统集成商（Tier 3）
- 合作模式：Revenue share 20-30%，ODE 提供培训、市场物料、共同营销
- 目标：Year 1 签约 3-5 个渠道伙伴，产生 $200K 管道

**AI 驱动 PLG**（AI Marketing 负责）：
- 工具栈：Claude（文案）、Midjourney（视觉）、Cursor（Landing Page 快速迭代）、Perplexity（竞品研究）
- 产出目标：5 件/周内容，2,000 月度 SEO 点击增量/月
- 渠道：LinkedIn + Twitter/X + YouTube + 技术博客

### 设计系统（Apple 风格）

参照 Apple Design System 原则应用于所有 ODE 营销和产品界面：

**核心设计原则：**
- **Typography**：SF Pro Display（标题）/ SF Pro Text（正文）/ SF Mono（代码）
- **Color**：接近纯白背景 `#FFFFFF` / `#F5F5F7`；文字 `#1D1D1F`；强调色 `#0071E3`（Apple Blue）→ 品牌适配为 ODE 深海蓝 `#0A2540`
- **Spacing**：8pt 网格，极致留白（每屏内容密度 ≤ 40%）
- **Imagery**：高质量工程渲染图，Cinema 4D 风格，白色/深色背景交替
- **Components**：圆角 12px，阴影极轻（`0 2px 8px rgba(0,0,0,0.08)`），无边框卡片
- **Motion**：缓入缓出，300ms，无炫技动画

## Further Notes

### 为什么是"达索"而非"Autodesk"？

达索是工程设计领域的终极参照——不是 CAD 公司，而是"工程知识管理"公司。CATIA、SIMULIA、ENOVIA、GENESYS 覆盖从概念到制造的完整工程生命周期。Autodesk 更偏消费级和制造业工具。ODE 的学术基础（Modelica、MBSE、多物理场）更接近达索的定位。

### AI Marketing 的工具栈建议

- 文案生成：Claude API / Claude Code
- 图片/渲染：Midjourney / Runway
- 视频剪辑：CapCut / Descript  
- SEO 研究：Perplexity + Semrush
- 落地页：Cursor + Framer
- 邮件自动化：Customer.io + AI 个性化
- 社交调度：Buffer / Hypefury

### 分销渠道优先级

1. **欧洲工程咨询公司**（德国/法国/荷兰），已有 Modelica 使用基础
2. **ANSYS/Dassault 经销商**，客户有迁移诉求
3. **大学技术转化中心**，低成本播种下一代用户
