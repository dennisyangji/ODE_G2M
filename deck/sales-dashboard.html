02 BANT+ Qualification Framework 5-point scoring · Hot / warm / cold segmentation
Scoring Rule
Score 5/5 = Hot (fast-track to demo) · Score 4/5 = Warm (qualify further) · Score 3/5 = Lukewarm (nurture 30 days) · Score <3 = Cold (remove from active pipeline)
BANT+ Criteria
B
Budget — Dedicated simulation / AI budget exists
Strong: "We have a CAD/simulation budget line" · "Our AI tools budget is approved"
Weak: "Need to find budget" · "I'd need to request it"
A
Authority — Speaking with decision maker or champion with access
Strong: VP Engineering / CTO / Managing Director in meeting
Weak: "I need to run this by my manager" without named access
N
Need — Active pain around simulation speed, tool fragmentation, or AI adoption
Strong: Mentions tool silos, slow iteration cycles, license cost burden
Weak: "Happy with current tools" · "No active projects requiring this"
T

Strong: "This quarter" · "Q2/Q3 rollout planned"
Weak: "Maybe next year" · "No immediate urgency"
+

Strong: Active Modelica user · FMI/FMU requirement · 5+ simulation engineers
Weak: No simulation experience · Excel-based analysis only

04 Objection Handling Map 8 critical objections · Validated responses
Objections 01–04
01 — "It's too expensive / we don't have budget."
ODE replaces 3+ tools (ANSYS + SolidWorks Simulation + CATIA). Show TCO: combined license cost vs. ODE Pro/Team. The ROI on license consolidation alone typically covers 3× the ODE cost. Start with the free tier — zero budget required.
02 — "We already use Dassault / Autodesk / ANSYS."
ODE works alongside existing tools via FMI export — no rip-and-replace required. Most customers run ODE in parallel for 60 days. Results speak. Propose a phase-in approach: use ODE for one specific workflow, measure the time savings, then decide.
03 — "Can we trust AI-generated simulation results?"
ODE AI generates standard, auditable Modelica code — not a black box. The engineer reviews and validates the model before running. Full audit trail. Every simulation output is reproducible and explainable. Your engineers remain in control.
04 — "We have security concerns about cloud."
Enterprise tier: SSO/SAML, role-based access, data residency (EU/US), full audit logs. SOC 2 Type II path in progress. On-premise deployment available for regulated industries. IT security review package provided immediately upon request.
Objections 05–08
05 — "We need to evaluate other alternatives first."
Encourage it. "Run the same benchmark in ODE and your current tool. We'll give you a free Pro trial for 30 days to make the comparison fair." Confidence in the product — no need to resist evaluation.
06 — "Our team isn't ready for AI in engineering workflows."
Start with free tier — zero-risk experimentation for one engineer. ODE doesn't replace engineer judgment — it eliminates the tedious setup work. The AI handles meshing and solver configuration; engineers handle the physics insight and decision-making.
07 — "My team won't adopt a new tool."
No installation required — runs in the browser right now. Onboarding is under 10 minutes. Bottom-up adoption: let one engineer try it on one project. When colleagues see simulation setup go from 3 days to 3 hours, adoption follows organically.
08 — "What if your company shuts down?"
All models are stored in standard Modelica format — fully portable. Export your entire project at any time. FMI/FMU output runs in any compliant solver. Zero vendor lock-in. Your engineering data belongs to you, always.
